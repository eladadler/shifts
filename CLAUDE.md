# הנחיות עבודה

- דחוף שינויים תמיד ישירות לענף `main` (commit + push ל-main), בלי ענפי עבודה נפרדים ובלי Pull Requests — אלא אם המשתמש ביקש אחרת במפורש.
- האפליקציה הראשית היא הקובץ `מערכת שיבוץ.html` שבשורש הריפו. תיקיית `project/` היא אב-טיפוס עיצובי (handoff) בלבד.
- אפליקציית העובדים (מובייל) היא הקובץ `עובד.html` בשורש הריפו. שתי האפליקציות מתממשקות דרך טבלאות Supabase משותפות: `app_employees`, `employee_requests`, `published_schedules` (ראו `supabase-setup.sql` ו-`הקמת-אזור-עובדים.md`). תיקיית `design-employee/` היא אב-טיפוס עיצובי בלבד.
